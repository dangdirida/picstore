# PicStore PRD

## 1. 개요
- **서비스 목적**: 누구나 디지털 이미지(그림/사진)를 업로드하고 판매할 수 있는 마켓플레이스
- **타겟 유저**: 작품을 판매하고 싶은 작가 / 이미지를 구매하고 싶은 구매자
- **핵심 가치**: 작가는 쉽게 올리고, 구매자는 쉽게 찾고 다운로드

---

## 2. 기능 목록

### 🔐 로그인 / 회원가입
- 이메일 + 비밀번호 회원가입 / 로그인
- 구글 소셜 로그인
- 로그아웃

### 🗄️ DB 설계
- 유저 테이블 (id, 이메일, 이름, 역할: 작가/구매자/어드민)
- 작품 테이블 (id, 제목, 설명, 가격, 무료/유료, 이미지 파일, 작가 id)
- 구매 테이블 (id, 유저 id, 작품 id, 결제 상태)
- 다운로드 테이블 (id, 유저 id, 작품 id, 날짜)
- 좋아요 테이블 (id, 유저 id, 작품 id)
- 댓글 테이블 (id, 유저 id, 작품 id, 내용, 날짜)

### 🎨 프런트엔드
- 메인 페이지 (작품 목록, 검색, 필터: 무료/유료)
- 작품 상세 페이지 (이미지 미리보기, 가격, 작가 정보, 다운로드/구매 버튼, 좋아요 버튼, 댓글 목록)
- 작가 프로필 페이지

### 👤 사용자 대시보드
- 내 구매 목록 / 다운로드 내역
- 내 업로드 작품 목록 (작가용)
- 작품 업로드 폼 (제목, 설명, 가격 설정, 이미지 첨부)
- 프로필 수정

### 🔗 네트워크 이펙트
- 작품 공유 버튼 (URL 복사, SNS 공유)
- 작가 팔로우 기능
- 인기 작품 섹션 (좋아요 수 기반)
- 신규 작품 섹션

### ⚙️ 백엔드 API
- 회원 인증 API
- 작품 업로드 / 수정 / 삭제 API
- 작품 목록 조회 / 검색 API
- 다운로드 API (무료는 바로, 유료는 결제 후)
- 좋아요 추가 / 취소 API
- 댓글 작성 / 삭제 API
- 결제 API (오늘은 UI만, 추후 Toss / Stripe 연동)

### 🚀 배포
- Vercel 배포
- 도메인 연결
- 환경변수 설정

### 🛡️ 어드민 페이지
- 전체 유저 목록
- 전체 작품 목록 / 삭제
- 신고된 작품 관리

### 🔒 보안
- 환경변수 분리
- 비인가 다운로드 방지 (유료 작품 URL 직접 접근 차단)
- API Rate Limiting
- 입력값 검증

### ✅ QA
- 회원가입 → 업로드 → 구매 → 다운로드 전체 흐름 테스트
- 모바일 반응형 확인
- 유료 작품 미결제 시 다운로드 차단 확인

## 3. 페이지 목록

| 페이지 | URL | 설명 |
|--------|-----|------|
| 메인 | `/` | 작품 목록, 검색 |
| 작품 상세 | `/works/[id]` | 이미지 상세, 구매/다운로드, 좋아요, 댓글 |
| 작가 프로필 | `/artist/[id]` | 작가 작품 모음 |
| 대시보드 | `/dashboard` | 내 업로드/구매 내역 |
| 업로드 | `/upload` | 작품 업로드 폼 |
| 어드민 | `/admin` | 관리자 페이지 |
| 로그인 | `/login` | 로그인/회원가입 |

---

## 4. 기술 스택
- **Frontend**: Next.js + Tailwind CSS
- **Backend**: Next.js API Routes
- **DB**: Supabase (PostgreSQL)
- **Auth**: Supabase Auth
- **Storage**: Supabase Storage (이미지 파일)
- **Deploy**: Vercel
